# YDown - 클라우드 지원 유튜브 오디오 추출기 (Max Quality)

이 프로젝트는 YouTube 영상 여러 개의 URL을 동시에 입력받아 **최상음질(320kbps)**의 오디오(음원)로 순차 추출하여 저장하는 웹 애플리케이션입니다.

## 🚀 새로운 기능
- **최상음질 적용**: 320kbps의 손실 없는 최고 품질의 음원(mp3, m4a, opus)을 보장합니다.
- **다중 다운로드(10개)**: 최대 10개의 유튜브 URL을 한 번에 입력하여 순서대로 다운로드 및 변환할 수 있습니다.
- **웹 다운로드 링크 제공**: 클라우드에 배포된 서버에서 작업이 끝나면, 사용자가 클릭하여 본인의 기기로 받을 수 있는 **다운로드 UI 버튼**이 팝업됩니다.

---

## ☁️ 웹앱 배포 안내 (Vercel vs Render)

유튜브 오디오 다운로드는 영상 다운로드와 FFmpeg(오디오 변환)이라는 무거운 작업이 함께 이루어집니다. 
따라서 **Vercel** 과 같은 Serverless(서버리스) 환경은 다음과 같은 이유로 **배포가 불가능에 가깝습니다.**
> 1. 서버리스는 최대 실행 시간이 10~60초로 매우 짧아 다운로드 도중 끊깁니다.
> 2. FFmpeg이 기본 설치되어 있지 않습니다.
> 3. 웹소켓과 임시 파일 시스템(다운로드 보관)의 제약이 큽니다.

### 👉 최적의 배포 방안 : Render.com (또는 Railway, Fly.io)
이 앱은 도커(Docker) 기반으로 완전히 최적화되어 있습니다. 무료 컨테이너 웹 호스팅을 지원하는 **[Render.com](https://render.com/)** 을 사용하는 것을 가장 추천합니다.

**💡 Render.com 배포 방법:**
1. 이 프로젝트(c:\dev\Ydown) 전체를 본인의 GitHub 저장소(Repository)에 업로드(Push)합니다.
2. Render.com 에 로그인하고 **[New +] -> [Web Service]** 를 클릭합니다.
3. 업로드한 GitHub 저장소를 연결합니다.
4. Render가 이 소스 안의 `Dockerfile`과 `render.yaml`을 자동 인식하여 FFmpeg 서버를 구축하고 무상으로 배포를 완료해줍니다!
5. 발급된 도메인(예: `ydown-audio.onrender.com`)으로 스마트폰이나 외부 PC에서 언제든 접속하여 오디오를 받을 수 있습니다.

## 💻 로컬(내 PC) 실행 방법

여전히 개발자의 로컬 PC에서도 완벽하게 동작합니다.

1. **파이썬 의존성 패키지 설치**
```bash
pip install -r requirements.txt
```

2. **서버 실행**
```bash
python app.py
```
👉 브라우저를 열고 `http://127.0.0.1:8000` 으로 접속하세요. (로컬에서는 추출된 파일이 자동으로 `c:\dev\Ydown\downloads` 폴더에도 저장되며, 웹에서도 다운로드 버튼이 뜹니다.)
